# Frontend and Admin API Contract

Base URL for integration:

- Public frontend: `NEXT_PUBLIC_API_BASE_URL` should point to `http://localhost:5000/api/v1` or equivalent.
- Admin app: use `VITE_API_BASE_URL` or equivalent pointing to `http://localhost:5000/api/v1`.

Response envelope:

```json
{
  "success": true,
  "message": "Request completed successfully",
  "data": {},
  "meta": {},
  "timestamp": "2026-05-09T00:00:00.000Z"
}
```

Error envelope:

```json
{
  "success": false,
  "message": "Error message",
  "data": null,
  "meta": null,
  "requestId": "request-id",
  "errors": []
}
```

## Integration Notes

- Protected anonymous/public flows should send either `Authorization: Bearer <accessToken>` or `X-SafeSpeak-Session: <anonymousSessionToken>`.
- Admin flows must use bearer tokens from `POST /auth/admin/login`.
- Do not call AI/RAG or ScamShield analysis until `process_with_ai=true`.
- Do not upload evidence until `cloud_sync=true`.
- Do not request warm referrals until `warm_referral=true`.
- Do not submit/share externally unless `share_with_agencies=true`.
- The frontend login helper currently uses an older "agent/profile" contract. Update it during frontend integration rather than changing backend response shape.
